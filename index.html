Getting Started with Auditrail

Welcome to the Auditrail documentation. This autonomous auditing platform is designed for enterprise AI deployments to detect, analyze, and repair logical discrepancies (hallucinations) in real-time.

Core Concepts

Claim Extraction

We process incoming text through a fine-tuned extraction layer to identify individual facts, statistics, and entities.

Autonomous Repair

When a hallucination is flagged, our rewriting model generates a semantically preserved, factually accurate replacement.

Architecture Overview

Auditrail employs a decoupled Client-Server architecture engineered for rapid inference and maximum security context execution.

  • Frontend: Rendered in highly optimized static assets (HTML/JS/CSS), utilizing dynamic SPA (Single Page Application) navigation methods for seamless state transitions without latency.
  • Execution Engine (Backend): A Flask-based orchestration layer that handles API key routing and manages payload telemetry.
  • Inference Clusters: External processing relies heavily on Anthropic's Claude framework for precision parsing, alongside integrated Groq hardware for ultra-low latency.

API Reference

The Auditrail REST API allows you to programmatically submit prompts for verification.

POST /v1/analyze

curl -X POST https://127.0.0.1:5000/analyze \
  -H "Content-Type: application/json" \
  -d '{"text": "Sample data.", "mode": "strict"}'

Endpoints & Configuration

Access the Security Configuration panel to manage system state. By default, the system requires a valid ANTHROPIC_API_KEY passed as an environment variable to the Flask backend. If left blank, the system defaults to a mocked hallucination state for development and testing purposes.

Troubleshooting

Q: The Execution Engine freezes at 'VERIFYING_FACTS...'.

A: Check the Flask console. If Anthropic is rate-limiting, the request may hang or return a 500 error. Check the Security panel Diagnostic Output.

Q: Why does my LocalStorage buffer say 0 Logs?

A: Run an audit first. Archives are populated strictly post-execution.
